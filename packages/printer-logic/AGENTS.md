# AGENTS.md - рабочий стандарт для `packages/printer-logic`

## 1. Область работы

Ты работаешь только с workspace-пакетом `packages/printer-logic` внутри monorepo `treed-shell`.

Назначение пакета:
- хранить общие TypeScript-типы состояния принтера и команд;
- хранить rules/capabilities для доступности действий;
- быть общим источником логики для `treed-shell` и будущего `treed-web-ui`.

По умолчанию режим работы: анализ, review типов/rules и осторожные точечные правки.

---

## 2. Приоритет правил

1. Явный запрос пользователя.
2. Безопасность и неразрушительность действий.
3. Инварианты этого файла.
4. Ограничения текущей задачи.
5. Локальные предпочтения среды и инструментов.

---

## 3. Язык и стиль

- Рабочий язык: русский.
- Отвечай коротко, технически точно, без воды.
- Не выдумывай контракт: проверяй публичный API и тесты.
- Не утверждай, что проверки запускались, если они не запускались.

---

## 4. Инварианты пакета

Нельзя добавлять:
- React, React hooks и UI-компоненты;
- CSS и визуальные токены;
- Tauri API;
- network fetch, Moonraker client или retry transport;
- shell/runtime-интеграцию.

Можно добавлять:
- чистые TypeScript-типы;
- pure functions для нормализации состояния;
- pure functions для rules/capabilities;
- unit-тесты на rules.

---

## 5. Источник истины

Источник истины в этом пакете:
- публичные exports из `src/index.ts`;
- unit-тесты;
- README с зафиксированными рисками и границами пакета.

Любое изменение rules должно сопровождаться unit-тестами. Если меняется публичный контракт или смысл блокировки, обнови README.

---

## 6. Проверки

Перед commit для изменений в `src/**` запускать:

```powershell
npm run typecheck
npm test
npm run build
```

Если проверка не запускалась, явно указать это в отчете.
