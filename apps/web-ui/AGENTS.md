# AGENTS.md - рабочий стандарт для `apps/web-ui`

## 1. Область работы

Ты работаешь только с workspace-папкой `apps/web-ui` внутри monorepo `treed-shell`.

Назначение папки:
- будущий веб-интерфейс TREED;
- отдельный UI-контур без Tauri;
- integration playground для `@treed/printer-logic` на первом этапе.

По умолчанию режим работы: анализ UI-задач, локализация экранов, осторожные правки веб-слоя.

---

## 2. Приоритет правил

1. Явный запрос пользователя.
2. Безопасность и неразрушительность действий.
3. Инварианты этого файла.
4. Ограничения текущей задачи.
5. Локальные предпочтения среды и инструментов.

---

## 3. Язык и стиль

- Рабочий язык: русский.
- Отвечай коротко и технически точно.
- Не утверждай, что проверки запускались, если они не запускались.
- Перед выводами проверяй фактический импорт `@treed/printer-logic` и текущий экран.

---

## 4. Инварианты папки

Нельзя:
- писать общую printer-логику в этой папке;
- дублировать rules/capabilities из `treed-shell`;
- добавлять Moonraker live transport без отдельной задачи;
- переносить Tauri API и runtime-обвязку;
- копировать компоненты/layout из `treed-shell` без явного решения.

Нужно:
- получать доступность действий через `@treed/printer-logic`;
- держать UI-слой отделенным от доменных rules;
- фиксировать mock/live предположения в README или задаче;
- сохранять доступность, адаптив и читаемость web UI.

---

## 5. Источник истины

- Общая логика: `packages/printer-logic` (`@treed/printer-logic`).
- UI вебморды: локальный React/Vite код `apps/web-ui`.
- Live transport и команды: не реализованы на первом этапе.

Если нужно изменить правило доступности действия, сначала менять `packages/printer-logic` и его тесты.

---

## 6. Проверки

Перед commit для изменений в `src/**` запускать:

```powershell
npm run typecheck
npm run build
```

Если проверка не запускалась, явно указать это в отчете.
